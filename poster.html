What ?

Nghiên cứu xây dựng hệ thống bảo mật lai (Hybrid Analysis) cho Kubernetes bằng cách kết hợp:

  • Giám sát thời gian thực từ Falco (Rule-based, Signature Detection)
  • Mô hình Học máy (Isolation Forest, One-Class SVM) để phát hiện hành vi bất thường (Anomaly Detection)
  • Cơ chế phản ứng tự động để cô lập các mối đe dọa
Why ?
  • Bối cảnh: Kubernetes là nền tảng tiêu chuẩn de-facto cho microservices nhưng tiềm ẩn rủi ro runtime (container escape, privilege escalation, zero-day).
  • Thực trạng: 87% image container chứa lỗ hổng nghiêm trọng (Sysdig 2023). NIST SP 800-190 nhấn mạnh bảo vệ tầng runtime là lớp phòng thủ quan trọng nhất.
  • Hạn chế: Giải pháp signature-based hiệu quả với mối đe dọa đã biết nhưng hạn chế với biến thể mới và zero-day. Falco có tỷ lệ cảnh báo giả cao.

Description
1. Mục tiêu Nghiên cứu
  • Tối ưu hóa độ chính xác: Kết hợp bộ luật tĩnh của Falco với mô hình ML để kiểm chứng chéo (cross-verify), giảm thiểu cảnh báo giả.
  • Phát hiện tấn công chưa biết: Ứng dụng Anomaly Detection để nhận diện hành vi lệch chuẩn, biến thể mới và zero-day.
  • Hiệu năng thời gian thực: Thiết kế pipeline xử lý luồng tối ưu, đảm bảo độ trễ thấp và tiêu thụ tài nguyên chấp nhận được.
2. Triển khai Hệ thống
  • Triển khai cluster Kubernetes thử nghiệm
  • Cài đặt và cấu hình Falco với eBPF driver
  • Xây dựng pipeline thu thập và xử lý dữ liệu (ETL)
  • Huấn luyện mô hình ML với dữ liệu runtime
  • Tích hợp Dashboard giám sát và cảnh báo
3. Kiến trúc Hệ thống
Hình 1. Mô hình kiến trúc tổng thể của hệ thống Hybrid Security
Hình 2. Mô hình luồng dữ liệu giữa các thành phần
4. Kết quả Mong đợi
  • Kiến trúc: Mô hình Hybrid Runtime Security với sơ đồ chi tiết.
  • Dataset: Bộ dữ liệu syscall từ workload hợp lệ và kịch bản tấn công MITRE ATT&CK.
  • Pipeline: Trích xuất đặc trưng với n-gram syscall và time-window statistics.
  • So sánh: Falco thuần vs Falco+ML - Dự kiến giảm đáng kể cảnh báo giả.
5. Chỉ số Đánh giá
  • Recall: Đảm bảo không bỏ sót tấn công
  • Precision: Mức độ giảm cảnh báo giả
  • F1-Score: Trung bình điều hòa
  • Detection Latency: Thời gian từ sự kiện đến cảnh báo
  • Resource Consumption: CPU/RAM overhead
